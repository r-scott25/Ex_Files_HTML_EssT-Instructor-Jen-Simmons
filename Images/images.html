Images

Image Tag/Element: <img>

Four Attributes of an Image Element

1. Source Attribute: src="img.jpg"
The source attribute tells the browser which image file to load.
2. Alternative Attribute: alt="black dog"
The alt attribute acts as a substitute for the image whenever the image can't be seen.
For example, people who are blind may use a screen reader where the ALT text is read aloud to them.
Some images don't need an explanation. Like a logo. In these cases, include the alt tag, but leave it empty.
If not, the file name for the image might get read aloud.
3. Width Attribue: width="400"
Let the browser know how wide the image is in pixels. We don't include the unit px. It's understood to mean pixels.
4. Height Attribute: height="300"
Let the browser know how tall the image is in pixels. We don't include the unit px. It's understood to mean pixels.
Why include the width and height if the image appears the size we want it to?
Once the file is already loaded and on the page, the width and height attributes don't do anything.
The browser figured out how big to display the image by downloading it, looking at the file and its characteristics,
and using that to calculate the layout. The problem is, if the browser has to get the size information out of the file,
then it can't calculate this part of the layout until after the image has downloaded. If instead we tell the browser how big each image is,
then it knows immediately and that gives the browser a split second head start in calculating the layout.
Including the width and height attributes will prevent a webpage from jumping around as images load, moving everything as you're trying to read.

Image Formats

To put an image on a webpage, it has to be in a file format that web browsers understand. There are several options.
We keep inventing new ones in a quest for the perfect balance between tiny file sizes and gorgeous-looking images.
We want as much data as possible, so that it'll look amazing.
But we want as little data as possible so images will download faster and not eat through people's data plans.

Goal: High Quality + Small File Size

Each fule format takes a different approach to solving this.
Each uses a different technique for compressing the image.

Four Main File Formats

  1. GIF
    • GIF = Graphics Interchange Format
    • The GIF is one of the oldest.
    • Good at compressing illustrations that have large areas of a single color
    • Limited color space of 256 colors (This can make photographs pixelated) - Bad choice for photographs
    • Can do transparency, with jagged edges
    • Can have multiple frames, and make a little movie
  2. SVG
    • SVG = Scalable Vector Graphic
    • Great for logos, icons, and other kinds of illustration.
    • Vector file (instead of pixels) - - It contains instructions for a drawing (e.g. Make a square, make it this color, put it here.
    • Because SVGs are made of vector and math, these images can scale to be physically quite large or very small without losing any quality. This makes the file size very small.
    • If you can use an SVG, you definitely should.
    • SVGs can be exported from Illustrator, Sketch, or other vector graphics programs
  3. JPG
    • Image format for compressing photos
    • Most digital cameras save photos that you shoot as JPG files.
    • When you put a JPG image on the web, make sure that either a human or robot resizes the image to an appropriate physical size
      and compresses it for the web. (Don't just stick a 5000 pixel wide, half-compressed JPG on your website.
      They are far too big for users to download and will make your site very slow.)
    • Can be compressed to much smaller sizes by simplifying the color info in the file.
    • Pick a balance between Size vs. Quality - - You can do that manually or rely on your web service to handle the task.
  4. PNG
    • Good solution when you have a photograph AND you need part of the photo to be transparent.
    • Good at compression illustrastions
    • Good at compressing photos and images
    • Depends on the use case - - - If you're manually compressing files, you should try several options and see which results in the smaller file.

Consider the physical size of an image, the file format and teh compressing settings when making your image files.
Aim to make them download as fast as possible.

Responsive Images

Displaying Images Based on the Resolution of the Screen

Sometimes, we want to display a big, beautiful, high resolution image for people with big screens and then,
shrink it way down for people on phones, on small screens. CSS can do this. One image file can be shown at any size.
But there's a problem. A big, high resolution image contains millions of pixels. And all that data can make the file size pretty big.
Which takes a lot of time to download on a slow network connection and can use up people's data plans and in some markets,
cost them a lot of money.

We don't want to deliver unneeded, high resolution data to small screens.

We don't want to deliver images that are physically smaller, have less color data, fewer pixels, and that are more compressed.
People with big desktop monitors will get a low quality photo blown up really big.

We can use the power of HTML to deliver different image files to different size screens.

We can make several different files and list them as a set of options in our HTML and let the browser decide which image to download.

It can do so in coordination with the operating system while taking the device hardware capabilities and network speed inot account as well.

More and more screens these days are not 1X screens. The retina screens, high density, DPI, super retina, etc. screens.
There are a bunch of scrrens with a pixel density that is 2X, 3X, 4X, where more data can be displayed by the screen.

The simplest way to support different screens is to create multiple copies of our image in different resolutions and then tell the browser
that those copies are available. The device can then decide what it wants to do. The browser will look at the screen density,
the network connection, the users settings and decide which image to use. Even is someone has a high resolution screen,
the browser might decide to download a lower resolution image.

I've created four copies of this photo at 480, and 960, 1440 and 1920 pixels wide.
Let me duplicate this basic HTML for displaying an image. So we can change the second one and compare the results.
We're going to still write the code for displaying the image in a normal way. This is good for older browsers.
We need the source attribute and we'll put our 1X version of the image here. We need the alt text just like before and the width and
the height. It will definitely still improve in performance, keep that. It's OK that the pixel numbers won't always be accurate.
The aspect ratio is what's important here. And 480 by 360 calculates to a four by three aspect ratio. All of these photos are
four by three. Now, let's add a source set attribute. Inside of it, we'll list the images that we are offering the browser as choices.
It's a comma separated list. With a URL to a file, a space, the resolution, like 2X, 3X, 4X, 1.5X, whatever, a comma, and the next one.
Now, the browser will swap out one version of this image for another based on what it thinks is best. This is a great solution for
providing different size images to handle retina and high DPI screens.

Responsive Width

Displaying Images Based on the Width of the Viewport/Size of the Screen (as well as the density)

The Size Attribute

Use the size attribute to list which size image to use at which media query.
The sizes attribute specifies for the browser how much of the viewport's width that image will take up at which break point

The srcset Attribute

The srsset attribute allowys you to specify multiple files to be used based on either viewport width or image pixel width.

Responsive Pictures

What if we want to use a different image on mobile than on desktop?
Maybe we want the image to be tall and narrow on mobile, where it's short and wide on desktop.
Maybe we want to use the full picture on desktop, but a cropped image on mobile.
Or even use a completely different photo altogether.

The image element alone, even with the srcset or size attributes can't do this.For this, we need the picture element

Picture Element

Wrap the image element with a picture element. With the picture wrapping the image element, we can list alternative options through the sources element. If you want to provide two options, create two source elements.

That's a lot of files to create and a lot of math involved.

On most sites, though, the work needed to publish each image in this way is automated.
A person who adds content to the site just uploads one large image. And then magical robots on the server do all the work of creating
the image files and writing out all the code. Those magical robots do need to be told what to do.

Figure and figcation

How to pair a caption for an image with the image itself.

Wrap the caption in a figcaption element, and wrap both the image and the caption together in a figure element.
This provides the browser with more semantic information about the content. Search engines in AI will now know
that these two pieces of content have a relationship to each other.

We can also use figure for things besides images, like an interactive graphic. We'll put it where the image element is in this markup.

The figure and figcaption are hand for anything that appears as a figure, illustrating something or anything that's a demonstration of a concept which needs a caption.
