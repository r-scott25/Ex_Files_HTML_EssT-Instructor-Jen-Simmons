Images

Image Tag/Element: <img>

Four Attributes of an Image Element

1. Source Attribute: src="img.jpg"
The source attribute tells the browser which image file to load.
2. Alternative Attribute: alt="black dog"
The alt attribute acts as a substitute for the image whenever the image can't be seen.
For example, people who are blind may use a screen reader where the ALT text is read aloud to them.
Some images don't need an explanation. Like a logo. In these cases, include the alt tag, but leave it empty.
If not, the file name for the image might get read aloud.
3. Width Attribue: width="400"
Let the browser know how wide the image is in pixels. We don't include the unit px. It's understood to mean pixels.
4. Height Attribute: height="300"
Let the browser know how tall the image is in pixels. We don't include the unit px. It's understood to mean pixels.
Why include the width and height if the image appears the size we want it to?
Once the file is already loaded and on the page, the width and height attributes don't do anything.
The browser figured out how big to display the image by downloading it, looking at the file and its characteristics,
and using that to calculate the layout. The problem is, if the browser has to get the size information out of the file,
then it can't calculate this part of the layout until after the image has downloaded. If instead we tell the browser how big each image is,
then it knows immediately and that gives the browser a split second head start in calculating the layout.
Including the width and height attributes will prevent a webpage from jumping around as images load, moving everything as you're trying to read.

Image Formats

To put an image on a webpage, it has to be in a file format that web browsers understand. There are several options.
We keep inventing new ones in a quest for the perfect balance between tiny file sizes and gorgeous-looking images.
We want as much data as possible, so that it'll look amazing.
But we want as little data as possible so images will download faster and not eat through people's data plans.

Goal: High Quality + Small File Size

Each fule format takes a different approach to solving this.
Each uses a different technique for compressing the image.

Four Main File Formats

  1. GIF
    • GIF = Graphics Interchange Format
    • The GIF is one of the oldest.
    • Good at compressing illustrations that have large areas of a single color
    • Limited color space of 256 colors (This can make photographs pixelated) - Bad choice for photographs
    • Can do transparency, with jagged edges
    • Can have multiple frames, and make a little movie
  2. SVG
    • SVG = Scalable Vector Graphic
    • Great for logos, icons, and other kinds of illustration.
    • Vector file (instead of pixels) - - It contains instructions for a drawing (e.g. Make a square, make it this color, put it here.
    • Because SVGs are made of vector and math, these images can scale to be physically quite large or very small without losing any quality. This makes the file size very small.
    • If you can use an SVG, you definitely should.
    • SVGs can be exported from Illustrator, Sketch, or other vector graphics programs
  3. JPG
    • Image format for compressing photos
    • Most digital cameras save photos that you shoot as JPG files.
    • When you put a JPG image on the web, make sure that either a human or robot resizes the image to an appropriate physical size
      and compresses it for the web. (Don't just stick a 5000 pixel wide, half-compressed JPG on your website.
      They are far too big for users to download and will make your site very slow.)
    • Can be compressed to much smaller sizes by simplifying the color info in the file.
    • Pick a balance between Size vs. Quality - - You can do that manually or rely on your web service to handle the task.
  4. PNG
    • Good solution when you have a photograph AND you need part of the photo to be transparent.
    • Good at compression illustrastions
    • Good at compressing photos and images
    • Depends on the use case - - - If you're manually compressing files, you should try several options and see which results in the smaller file.

Consider the physical size of an image, the file format and teh compressing settings when making your image files.
Aim to make them download as fast as possible.
